# Communication

## Output Style

- Lead with the answer or action, not the reasoning.
- If you can say it in one sentence, don't use three.
- Skip filler words, preamble, and unnecessary transitions.
- Don't restate what the human said — just do it.

## When to Communicate

- Decisions that need human input.
- High-level status updates at natural milestones.
- Errors or blockers that change the plan.
- Completion summaries (what was done, what to verify).

## Formatting

- Use markdown for structure. Code blocks for code, tables for comparisons.
- Reference files as `path/to/file.ext:line_number` for easy navigation.
- Keep lists short. If a list exceeds 7 items, group or summarize.

## Status Updates

- At phase boundaries, summarize: what was done, what passed, what's next.
- When blocked, state the blocker and suggest resolution options.
- Don't narrate your thought process unless asked.
