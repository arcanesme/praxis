# PRAXIS CLI

> Disciplined action. Theory without action is idle; action without theory is reckless.

A Python CLI that scaffolds, configures, and validates the **PRAXIS protocol** вЂ” a vendor-agnostic, context-driven development workflow that works identically across **Claude Code** and **OpenAI Codex**.

## Install

```bash
pipx install praxis-cli
```

## Quick Start

```bash
# One-time: set global preferences
praxis config

# Per-project: scaffold PRAXIS
cd your-project
praxis init

# Validate everything
praxis doctor
```

Then open your AI tool and run:

| Claude Code | OpenAI Codex |
|-------------|--------------|
| `/setup` | `praxis setup` |
| `/new-track` | `praxis new-track` |
| `/implement` | `praxis implement` |
| `/review` | `praxis review` |
| `/status` | `praxis status` |
| `/verify` | `praxis verify` |
| `/commit-push-pr` | `praxis commit` |
| `/simplify` | `praxis simplify` |
| `/sync-context` | `praxis sync` |
| `/deploy-preview` | `praxis deploy` |

## CLI Commands

| Command | What it does |
|---------|-------------|
| `praxis init` | Scaffold PRAXIS.md, commands, verification config, and tool-specific files |
| `praxis config` | Set global preferences, sync to ~/.claude/CLAUDE.md |
| `praxis doctor` | Validate project setup, tool sync, and verification tools |
| `praxis bootstrap` | Regenerate tool-specific files after updating PRAXIS.md |
| `praxis status` | Show all tracks with Rich-formatted progress tables |
| `praxis verify` | Run enabled checks from praxis/verification.md (quick/full/pr modes) |

## How It Works

**PRAXIS.md** is the single source of truth. It defines:
- Protocol rules (ask before building, spec before code, phase gates)
- Track types (code, architecture, federal, content)
- Spec, plan, and review formats
- Verification requirements

**`praxis init`** scaffolds the full structure and generates tool-specific instruction files:
- `CLAUDE.md` + `.claude/commands/` for Claude Code
- `AGENTS.md` for OpenAI Codex
- `.github/workflows/praxis-pr-review.yml` for automated Codex PR reviews

**`praxis config`** sets global preferences that sync across all tools:
- Always ask clarifying questions before building
- Include recommendations in every option
- Enable/disable specific AI tools

**`praxis doctor`** validates everything is wired up correctly.
